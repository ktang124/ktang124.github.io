Is Beauty Really Subjective?

By Shiv Medler, Luke Luo, Kevin Tang

Motivation: Humans can recognize beauty instantly, even babies see beauty. However, it is difficult to identify what specifically makes someone beautiful or not beautiful. Why are some features desirable but others are not? We know that there are subjective preferences rooted in culture. Some may generalize these preferences into western and eastern beauty standards. For this project, we will try to see if there are any universal features of beauty.

One data source that we can use for this study is the Chicago Face Database. This database is a folder that contains about 600 images of faces from the United States. These faces include men and women of white, latin, asian, and black descent. For each of these faces, there are corresponding measurements of different attributes of the face alongside subjective ratings of the face based on a U.S. voter sample. We will do regression on the attribute measurements and deep learning on the face images to see if the machine can find variables that explain the variation in the subjective ratings of the faces. If these kinds of variables exist, then we know that there are objective factors to beauty.

One application of this study lies in the face's ability to indicate the health of the individual. Features like acne are unattractive because irregularities in skin usually indicate disease like smallpox or chickenpox. Some features indicate underlying conditions like fetal alcohol syndrome, whose characteristics can be recognized in children. The idea that attractiveness implies health and unattractiveness implies lack of health lends itself to an application for the beauty algorithm. It could be used to detect certain features that suggest diseases and conditions early on.

Furthermore, we know that beauty measurement algorithms are prevalent in today's social media apps, particularly tiktok, where more attractive content creators are pushed to users' recommendations. Nonetheless, these algorithms are used because they attract users' attention. More attention drives more ad revenue, more ad revenue drives more profits for social media companies. The beauty algorithm is just one tool used in recommendation systems. Therefore, through our dataset, we can gain insight into these systems.

The issue now is with the column names since the names of the columns were distributed across two rows. This leaves us with three distinct possibilites:

  1. The column name is only present in the dataframe column name and the first row of that column is marked as not a number.

  2. The actual column name is present in the first row of the column and the column name in the dataframe.

  3. The pandas dataframe column name is marked as unnamed while the actual name is in the first row of the column data.

In the code below, I renamed the pandas dataframe taking into account these three cases. I did this by creating the new_names dictionary and iterating through the column name and data in the dataframe using the pandas iteritems method. This is because the pandas rename takes in a dictionary with the old names being assigned to the new ones. For the first case, I kept the old name. With the second case, I concantenated the name and the string value for the first row in the column. For the third case, I set the new column to the first row of the column. The result of this renaming is shown below.

Variable Selection:

For the purposes of this project, we will be focusing on how the measured attributes of the peoples' faces would affect whether people would view them as attractive. Hence, in accordance to the lookup table, we would first look for the column labels that have a corresponding VarID that begins with P. These will be our predictors. Then, we will use Attractive R013_mean as the response variable. The variables measures on average, from a scale of 1-7, how a sample of Americans rated a face as attractive.

The major measurements we lose from selecting the predictors in this way would be the perceptions of the people rating the faces in terms of attractiveness. An example would be the perception of race and gender of a given face for the people rating the face. However, since we are focusing on the unconscious features that people can focus on in assessing a person's beauty, we are not going to look at these features that people were consciously thinking about when assessing the face's attractiveness. Another measurement we would not include would be the size of the sample of people that rated the faces' attractiveness. This is because we believe that these sample sizes were roughly the same in which this information would not be useful in describing the variation for attractiveness. Finally, we are also not looking at the gender and race of the individual since according to the description of Attractive R013-mean, the participants were asked to rate people's face with respect to other people of the same race and gender. Hence, these two variables were controlled for in the data collection process.

To remove the unwanted variables, we used the lookup table shown before in which we intented to use the column's associated VarID in deciding to delete them. Model was removed since it had no corresponding VarID in the table and was not needed since it only stored the filename for a given photo.

Similar to creating the face dataframe, we first removed the grey description in excel before using the read_excel method to read the data. After putting the lookup_table into a pandas dataframe, we iterated through all of the columns in the face dataframe. We extracted the first word for each of the column names such that it can match up with the label listed in the lookup table. We then filtered the dataframe to include the entries that had this label as VarLabel. Multiple entries may be returned in this case, but we knew that they would all start with the same letter for the VarID. Hence, we only needed to look at the first entry. If the first letter for the VarID was R and the variable was not the Attractive measure, we would remove it since we are only looking at the Attractive subjective measure. The columns to be removed were stored in the dropped_columns list and removed by the pandas dropped method.

Addressing Multicollinearity: One issue that is present with these variables is multicollinearity. This is where the predictor variables are correlated with each other. This can cause high variance in the estimated coefficients for these variables and lead to misleading interpretations of the regression results. As seen with the column names, there are variables like the measurements of the eye width and height that would seem to be correlated. To see if any of the variables cause this issue, we would look at the Variance Influence Factor values for them. These values are calculated for each predictor by seeing how much of the variation in this predictor is explained by the rest of these predictors.

More information on Multicollinearity and VIF is explained in this link

X_variables holds all the measurement predictors for the model which is why the Attractive variable was dropped to form it.

For the calculate_vif function, the add_constant method from statsmodels is first used since when fitting the linear regression, we plan to include a constant term such that the line can better fit the data. Then, a new dataframe is created in which the feature column would hold the column names while the VIF column would hold the corresponding VIF values. The VIF was calculated by using the variance_inflation_factor method from statsmodel which takes in a dataset of explanatory variables and an index to calculate a VIF value for a given variable. In the code below, this value is calculate for all the indices in the column and stored in a list. This list is then used as the column data for the VIF column. The resulting dataframe is then returned.

In [ ]:
def calculate_vif(X):
  X = add_constant(X)
  vif_data = pd.DataFrame()
  vif_data["feature"] = X.columns
  vif_data["VIF"] = [variance_inflation_factor(X.values, i) for i in range(len(X.columns))]
  return vif_data

The output of the calculate_vif function on the X_variables is shown below. A VIF value over 10 indicates multicollinearity in which the variation in that variable can be significantly explained by another explanatory variable. The usual case is that faces are mostly symmetrical so a measurement for the left and right part of the face (ex. left and right eye) are likely to be the same. There are also some measurements that are ratios between 0-1 which we can remove since they are calculations made based on other measurements already present (ex. we can remove forehead to face height ratio since we have forehead and face height measurements). As seen in the output, all of the variables have VIF values above 10. Hence, more variable preprocessing needs to be done to address this effect.

In the code below, we are finding the explanatory variables that will be used to predict attractiveness taking into account multicollinearity. This is done by iterating through the columns and appending each variable to the final columns list. A local dataframe is constructed with the new_df variable by specifying the variables in this list when accessing the X_variables dataframe. When a new variable is added, if this causes one of the variables to have a VIF value of over 10, then this variable is removed and will not be a part of the final_columns list. In the VIF value condition, iloc is used since we will only look at the VIF values for the explanatory variables rather than including the constant. The final_columns list is then outputted and shows the columns that were able to be included in the model without having one of the explanatory values have a VIF value above 10.

Description: Predicting attractiveness by looking at images. Look at the images in the folder CFD. The names of the images indicate both the gender and race.

Firstly, it will take much more resources than we have access to in order to train a model from scratch. That is why for this project, we will be outsourcing a pre-initialized image recognition model by using Transfer Learning.

VGG16 is a pre-trained CNN transfer learning framework that has 138 million parameters. It contains a model, VGG_Face, that is tuned for facial recognition. More information about VGG16 can be found here: https://keras.io/api/applications/vgg/

Here we load in the VGG model, with the top fully connected layer removed so that we can train it on our own dataset. Additionally, we add a several convolutional layers to the very bottom of the model in order to narrow down the 2622 classifications to a single output, the attractiveness score between 0 and 7.
